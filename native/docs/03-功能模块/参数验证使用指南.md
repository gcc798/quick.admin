# 参数验证使用指南

本文档介绍项目中参数验证的多种使用方式，包括基础验证、自定义错误提示、场景化提示等。

---

## 一、基础用法

### 1.1 在请求结构体中定义验证规则

使用 `binding` 标签定义验证规则：

```go
package request

// CreateUserRequest 创建用户请求
type CreateUserRequest struct {
    UserName    string `json:"userName" binding:"required,min=3,max=20"`
    NickName    string `json:"nickName" binding:"required"`
    Password    string `json:"password" binding:"required,min=6"`
    Email       string `json:"email" binding:"omitempty,email"`
    Phonenumber string `json:"phonenumber" binding:"omitempty,len=11"`
    Sex         int32  `json:"sex" binding:"omitempty,oneof=0 1 2"`
    Status      int32  `json:"status" binding:"omitempty,oneof=0 1"`
}
```

### 1.2 在 Controller 中使用

```go
func (h *userController) Create(c *gin.Context) {
    var req request.CreateUserRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        // 使用 Translate 翻译错误
        response.FailCode(c, 400, validator.Translate(err))
        return
    }
    // ... 业务逻辑
}
```

### 1.3 默认错误提示效果

| 验证规则 | 输入值 | 错误提示 |
|---------|--------|---------|
| `required` | 空字符串 | `用户名不能为空` |
| `min=3` | "a" | `用户名长度不能少于3个字符` |
| `max=20` | 超长字符串 | `用户名长度不能超过20个字符` |
| `email` | "abc" | `邮箱格式不正确` |
| `len=11` | "123" | `手机号长度必须为11位` |
| `oneof=0 1 2` | 3 | `性别的值必须是以下之一: 0 1 2` |

## 二、常用验证标签

### 2.1 字符串验证

```go
type StringDemo struct {
    // 必填，长度 3-20
    UserName string `binding:"required,min=3,max=20"`
    
    // 固定长度 11（如手机号）
    Phone string `binding:"len=11"`
    
    // 选填，如果填写必须符合邮箱格式
    Email string `binding:"omitempty,email"`
    
    // 字母和数字
    Code string `binding:"alphanum"`
    
    // 只允许字母
    Name string `binding:"alpha"`
    
    // 只允许数字
    Age string `binding:"numeric"`
    
    // 正则表达式
    IDCard string `binding:"len=18"` // 可配合自定义验证器
}
```

### 2.2 数值验证

```go
type NumberDemo struct {
    // 必填，大于 0
    Age int `binding:"required,gt=0"`
    
    // 大于等于 1，小于等于 100
    Score int `binding:"gte=1,lte=100"`
    
    // 等于 5
    Level int `binding:"eq=5"`
    
    // 不等于 0
    Status int `binding:"ne=0"`
}
```

### 2.3 其他验证

```go
type OtherDemo struct {
    // URL 格式
    Website string `binding:"url"`
    
    // IP 地址
    IP string `binding:"ip"`
    
    // 日期时间格式
    BirthDate string `binding:"datetime=2006-01-02"`
    
    // 包含在列表中
    Color string `binding:"oneof=red green blue"`
    
    // 枚举值（数字）
    Status int32 `binding:"oneof=0 1 2"`
    
    // 嵌套结构体验证
    Address AddressRequest `binding:"dive"`
}
```

### 2.4 组合规则

```go
type CombinedDemo struct {
    // 选填，但如果填了必须符合邮箱格式且长度不超过100
    Email string `binding:"omitempty,email,max=100"`
    
    // 手机号：选填，必须是11位数字
    Phone string `binding:"omitempty,len=11,numeric"`
    
    // 密码：必填，6-20位
    Password string `binding:"required,min=6,max=20"`
}
```

## 三、自定义错误提示（msg 标签）

### 3.1 场景问题

同名字段在不同场景需要不同提示：
- **创建用户**：`email` 是必填的，提示"请输入注册邮箱"
- **更新用户**：`email` 是选填的，提示"邮箱格式不正确"

### 3.2 使用 msg 标签

```go
// 创建场景
type CreateUserRequest struct {
    Email string `json:"email" binding:"required,email" msg:"注册时请输入有效的邮箱地址"`
}

// 更新场景
type UpdateUserRequest struct {
    Email string `json:"email" binding:"omitempty,email" msg:"邮箱格式不正确，请检查"`
}
```

### 3.3 Controller 中使用

```go
func (h *userController) Create(c *gin.Context) {
    var req request.CreateUserRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        // 使用 TranslateWithMsg 优先读取 msg 标签
        response.FailCode(c, 400, validator.TranslateWithMsg(err, &req))
        return
    }
}
```

### 3.4 效果对比

| 场景 | 使用 Translate | 使用 TranslateWithMsg |
|------|---------------|---------------------|
| 创建用户 | `邮箱不能为空` | `注册时请输入有效的邮箱地址` |
| 更新用户 | `邮箱格式不正确` | `邮箱格式不正确，请检查` |

### 3.5 完整示例

```go
type CreateUserRequest struct {
    UserName    string `json:"userName" binding:"required,min=3,max=20" msg:"用户名必须是3-20个字符"`
    NickName    string `json:"nickName" binding:"required" msg:"请输入昵称"`
    Password    string `json:"password" binding:"required,min=6" msg:"密码长度不能少于6位"`
    Email       string `json:"email" binding:"omitempty,email" msg:"邮箱格式不正确，请输入有效的邮箱地址"`
    Phonenumber string `json:"phonenumber" binding:"omitempty,len=11" msg:"手机号必须是11位数字"`
}
```

错误提示：
```json
{
    "code": 400,
    "msg": "用户名必须是3-20个字符；邮箱格式不正确，请输入有效的邮箱地址",
    "data": null
}
```

## 四、API 函数说明

### 4.1 Translate

使用默认的中文翻译器翻译错误信息。

```go
func Translate(err error) string
```

**特点**：
- 自动识别字段名（json tag）
- 使用预设的中文提示模板
- 无需额外配置

### 4.2 TranslateWithMsg

优先使用结构体的 `msg` 标签，如果没有则回退到默认翻译。

```go
func TranslateWithMsg(err error, obj interface{}) string
```

**特点**：
- 支持同名字段不同提示
- 通过反射读取 msg 标签
- 需要传入具体类型指针（不能是 interface{}）

### 4.3 对比

| 函数 | 优先级 | 适用场景 |
|------|--------|---------|
| `Translate` | 默认翻译 | 通用场景，字段提示统一 |
| `TranslateWithMsg` | msg 标签 > 默认翻译 | 同名字段不同提示的场景 |

## 五、最佳实践

### 5.1 推荐做法

```go
// 1. 基础验证用默认翻译
response.FailCode(c, 400, validator.Translate(err))

// 2. 需要场景化提示时用 msg 标签
response.FailCode(c, 400, validator.TranslateWithMsg(err, &req))
```

### 5.2 字段命名规范

```go
// ✅ 正确：使用 json tag 作为字段标识
type User struct {
    UserName string `json:"userName" binding:"required" msg:"用户名不能为空"`
}

// ❌ 错误：避免使用大驼峰作为 json tag
type User struct {
    UserName string `json:"UserName" binding:"required"`  // 不推荐
}
```

### 5.3 msg 标签编写建议

```go
// ✅ 好的提示：具体、有指导性
Email string `msg:"请输入有效的企业邮箱（如：name@company.com）"`

// ❌ 差的提示：太简单或太技术
Email string `msg:"email error"`           // 不清晰
Email string `msg:"Field validation for Email failed"` // 太技术
```

### 5.4 常见错误处理

```go
func (h *userController) Create(c *gin.Context) {
    var req request.CreateUserRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        // 优先尝试 TranslateWithMsg，如果不支持则回退到 Translate
        msg := validator.TranslateWithMsg(err, &req)
        if msg == "" {
            msg = validator.Translate(err)
        }
        response.FailCode(c, 400, msg)
        return
    }
}
```
