## 🚀 部署步骤

### 方式一：直接部署

```bash
# 1. 编译应用
GOOS=linux GOARCH=amd64 go build -o bin/api cmd/api/main.go

# 2. 上传文件到服务器
scp bin/api user@server:/app/
scp cmd/api/conf.prod.yaml user@server:/app/
scp cmd/api/*.yaml user@server:/app/

# 3. 在服务器上运行
ssh user@server
cd /app
./api
```

### 方式二：Docker 部署

```bash
# 1. 构建镜像
docker build -t quick.admin:1.0.0 .

# 2. 推送到仓库
docker tag quick.admin:1.0.0 registry.example.com/quick.admin:1.0.0
docker push registry.example.com/quick.admin:1.0.0

# 3. 在服务器上运行
docker pull registry.example.com/quick.admin:1.0.0
docker run -d \
  --name quick.admin \
  -p 8080:8080 \
  -v /path/to/config:/app/config \
  registry.example.com/quick.admin:1.0.0
```

### 方式三：Kubernetes 部署

```bash
# 1. 创建命名空间
kubectl create namespace quick.admin

# 2. 创建 Secret
kubectl create secret generic quick.admin-config \
  --from-file=conf.prod.yaml \
  -n quick.admin

# 3. 部署应用
kubectl apply -f k8s/deployment.yaml -n quick.admin
kubectl apply -f k8s/service.yaml -n quick.admin
kubectl apply -f k8s/ingress.yaml -n quick.admin

# 4. 检查部署状态
kubectl get pods -n quick.admin -w
kubectl describe pod <pod-name> -n quick.admin
```

## 🔧 故障排查

### 服务无法启动

**检查项：**
1. 配置文件是否正确
2. 数据库连接是否正常
3. Redis 连接是否正常
4. 端口是否被占用
5. 日志文件权限是否正确

**排查命令：**
```bash
# 查看日志
tail -f logs/app.log

# 检查端口
netstat -tlnp | grep 8080

# 测试数据库连接
psql -h <host> -U <user> -d <db>

# 测试 Redis 连接
redis-cli -h <host> ping
```

### 健康检查失败

**检查项：**
1. 数据库是否可访问
2. Redis 是否可访问
3. 网络策略是否正确
4. 防火墙规则是否正确

**排查命令：**
```bash
# 查看 Pod 日志
kubectl logs <pod-name> -n quick.admin

# 查看 Pod 事件
kubectl describe pod <pod-name> -n quick.admin

# 进入容器调试
kubectl exec -it <pod-name> -n quick.admin -- /bin/sh
```

### 优雅关闭失败

**检查项：**
1. terminationGracePeriodSeconds 是否足够
2. 是否有长时间运行的请求
3. 资源清理是否正常

**排查命令：**
```bash
# 查看容器退出日志
docker logs <container-id>

# 查看 Pod 删除过程
kubectl delete pod <pod-name> -n quick.admin --v=8
```
